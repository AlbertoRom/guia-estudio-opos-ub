Mes 1 · Bloc 1
Programació orientada a objectes

La programació orientada a objectes (POO) és un paradigma que organitza el codi al voltant d'objectes — entitats que combinen dades (atributs) i comportament (mètodes). És la base de Java, JavaScript, PHP i la majoria de tecnologies del temari.

Els quatre pilars de la POO

1. Encapsulament

Consisteix a ocultar la implementació interna d'un objecte i exposar només el que és necessari. Es fa amb modificadors d'accés: private, protected, public.

public class CompteBancari {
    private double saldo; // ocult a l'exterior

    public double getSaldo() { return saldo; }

    public void ingressar(double import) {
        if (import > 0) saldo += import; // validació interna
    }
}
💡

L'encapsulament permet canviar la implementació interna sense trencar el codi extern. És la base del disseny robust.

2. Herència

Permet que una classe hereti atributs i mètodes d'una altra. La classe filla pot sobreescriure (@Override) comportament del pare. A Java, l'herència és simple (una sola classe pare).

public class Animal {
    protected String nom;
    public void parlar() { System.out.println("..."); }
}

public class Gos extends Animal {
    @Override
    public void parlar() { System.out.println("Bup!"); }
}

3. Polimorfisme

Capacitat d'usar una referència de tipus pare per a referenciar objectes de classes filles. Hi ha dos tipus: en temps de compilació (sobrecàrrega de mètodes) i en temps d'execució (sobreescriptura).

Animal a = new Gos(); // referència Animal, objecte Gos
a.parlar(); // crida "Bup!" — polimorfisme en execució

4. Abstracció

Definir contractes sense implementació. A Java es fa amb abstract class i interface.

CaracterísticaClasse abstractaInterfície
InstanciableNoNo
Herència múltipleNo (extends 1)Sí (implements N)
Pot tenir atributsSíNomés constants (Java 8+: default methods)
Quan usarCompartir codi base parcialDefinir contracte pur

Principis SOLID

  • S – Single Responsibility: una classe, una responsabilitat.
  • O – Open/Closed: obert a extensió, tancat a modificació.
  • L – Liskov Substitution: les subclasses han de poder substituir la classe pare sense errors.
  • I – Interface Segregation: interfícies específiques millor que una de genèrica.
  • D – Dependency Inversion: dependre d'abstraccions, no d'implementacions concretes.

Patrons de disseny comuns

Singleton

Garanteix que una classe tingui una sola instància en tota l'aplicació.

public class Configuracio {
    private static Configuracio instancia;
    private Configuracio() {}
    public static Configuracio getInstance() {
        if (instancia == null) instancia = new Configuracio();
        return instancia;
    }
}

Factory

Delega la creació d'objectes a un mètode, sense especificar la classe concreta al codi client.

Observer

Un objecte (subjecte) notifica automàticament els seus observadors quan canvia d'estat. És la base de molts frameworks reactius (RxJS, Angular).

Lambdes i Streams (Java 8+)

Els streams permeten processar col·leccions de forma declarativa i funcional, semblant a SQL.

List<String> noms = Arrays.asList("Anna", "Bernat", "Clara", "David");

List<String> resultat = noms.stream()
    .filter(n -> n.length() > 4)   // filtra per longitud
    .map(String::toUpperCase)         // transforma a majúscules
    .sorted()                          // ordena
    .collect(Collectors.toList());    // recull resultat
// Resultat: [BERNAT, CLARA, DAVID]

Operacions de Stream

  • Intermediàries (lazy): filter(), map(), flatMap(), sorted(), distinct(), limit()
  • Terminals (eager): collect(), forEach(), count(), reduce(), findFirst(), anyMatch()

Optional

Optional<T> evita el NullPointerException forçant el tractament explícit d'un valor potencialment nul.

Optional<String> opt = Optional.ofNullable(getBD());
String valor = opt.orElse("per defecte");
opt.ifPresent(v -> System.out.println(v));

Concurrència

  • Thread: unitat bàsica d'execució concurrent.
  • synchronized: garanteix que un bloc de codi només l'executa un thread alhora.
  • ExecutorService: gestió de pools de threads. Preferit a crear Thread manualment.
  • Future / CompletableFuture: resultat asíncron d'una operació.
ExecutorService pool = Executors.newFixedThreadPool(4);
Future<Integer> futur = pool.submit(() -> {
    // càlcul costós
    return 42;
});
int resultat = futur.get(); // bloqueja fins que acabi

  • JPA (Java Persistence API): ORM estàndard per mapear objectes Java a bases de dades relacionals.
  • CDI (Context and Dependency Injection): injecció de dependències estàndard Java EE.
  • JAX-RS: API per construir serveis REST a Java.

JavaScript i TypeScript

JavaScript és el llenguatge de la web. TypeScript n'és un superconjunt amb tipatge estàtic opcional que compila a JS. Ambdós s'usen àmpliament en el frontend i el backend (Node.js).

Conceptes clau de JavaScript

Closures

Una funció "recorda" el seu entorn léxic (les variables del scope on va ser creada), fins i tot quan s'executa fora d'aquest.

function comptador() {
  let n = 0;
  return () => ++n; // closure: recorda n
}
const c = comptador();
c(); // 1
c(); // 2

Asincronisme: Promises i async/await

// Amb Promises
fetch('/api/dades')
  .then(res => res.json())
  .then(dades => console.log(dades))
  .catch(err => console.error(err));

// Equivalent amb async/await (més llegible)
async function carregarDades() {
  try {
    const res = await fetch('/api/dades');
    const dades = await res.json();
    return dades;
  } catch (err) {
    console.error(err);
  }
}

Mes 1 · Bloc 5
Bases de dades: SQL i Oracle

El model relacional organitza les dades en taules amb relacions entre elles. SQL és el llenguatge estàndard per consultar i manipular aquestes dades. Oracle és el SGBD (sistema gestor de base de dades) principal de la UB.

Model relacional

  • Taula (relació): conjunt de files (tuples) i columnes (atributs).
  • Clau primària (PK): identifica unívocament cada fila.
  • Clau forana (FK): referència a la PK d'una altra taula (integritat referencial).
  • Normalització: procés d'organitzar les dades per eliminar redundàncies. Formes normals: 1FN, 2FN, 3FN, BCNF.

SQL: categories de sentències

CategoriaSentènciesPropòsit
DDLCREATE, ALTER, DROP, TRUNCATEDefinir estructura
DMLSELECT, INSERT, UPDATE, DELETEManipular dades
DCLGRANT, REVOKEControl d'accés
TCLCOMMIT, ROLLBACK, SAVEPOINTTransaccions

JOINs

-- INNER JOIN: només files amb correspondència en ambdues taules
SELECT u.nom, c.titol
FROM usuaris u
INNER JOIN cursos c ON u.curs_id = c.id;

-- LEFT JOIN: totes les files de l'esquerra, NULLs a la dreta si no hi ha correspondència
SELECT u.nom, c.titol
FROM usuaris u
LEFT JOIN cursos c ON u.curs_id = c.id;

Subconsultes i funcions d'agregació

-- Subconsulta: usuaris amb sou superior a la mitjana
SELECT nom FROM empleats
WHERE sou > (SELECT AVG(sou) FROM empleats);

-- Agregació per departament
SELECT departament, COUNT(*), AVG(sou)
FROM empleats
GROUP BY departament
HAVING COUNT(*) > 5;  -- HAVING filtra grups, WHERE filtra files

Oracle: PL/SQL

PL/SQL és l'extensió procedural d'Oracle que permet escriure programes complets dins la BD.

-- Procediment emmagatzemat
CREATE OR REPLACE PROCEDURE augmentar_sou(
  p_dept IN VARCHAR2,
  p_pct  IN NUMBER
) IS
BEGIN
  UPDATE empleats
  SET sou = sou * (1 + p_pct/100)
  WHERE departament = p_dept;
  COMMIT;
END;

Índexs i rendiment

  • Índex B-Tree: el més comú. Ideal per cerques per igualtat i rang.
  • Índex Bitmap: Oracle. Per columnes amb poca cardinalitat (p.ex. sexe, estat).
  • EXPLAIN PLAN: analitza com Oracle executa una consulta.

Seguretat: SSO i protocols d'autenticació

L'autenticació és el procés de verificar qui ets. L'autorització determina què pots fer. En entorns corporatius com la UB s'usa SSO per autenticar l'usuari una sola vegada i accedir a múltiples serveis.

SSO (Single Sign-On)

Permet a un usuari autenticar-se una vegada i accedir a múltiples aplicacions sense tornar a introduir credencials. Conceptes clau:

  • IdP (Identity Provider): servidor que autentica l'usuari i emet tokens/assertions.
  • SP (Service Provider): aplicació que confia en l'IdP per verificar la identitat.
  • Token: prova digital d'autenticació (JWT, SAML assertion, etc.).

CAS (Central Authentication Service)

Protocol SSO creat per la Universitat de Yale. Molt usat en entorns universitaris.

  1. L'usuari accedeix al SP. CAS redirigeix al login de CAS.
  2. L'usuari s'autentica al servidor CAS.
  3. CAS emet un Service Ticket (ST) i redirigeix al SP.
  4. El SP valida el ST directament contra el servidor CAS (back-channel).
  5. CAS confirma la identitat. Accés concedit.

SAML 2.0

Estàndard XML per intercanviar dades d'autenticació i autorització. Usat en entorns empresarials.

  • SAML Assertion: document XML signat digitalment que conté informació de l'usuari.
  • Binding: mecanisme de transport (HTTP Redirect, HTTP POST).
  • Metadata: configuració de confiança entre IdP i SP.
💡

SAML és a XML el que OpenID Connect és a JSON. SAML és més antic i verbós; OpenID Connect és més modern i lleuger.

OAuth 2.0

Protocol d'autorització (no autenticació) que permet a una aplicació accedir a recursos en nom d'un usuari sense compartir la contrasenya.

Grant TypeÚsQuan usar
Authorization CodeApps web amb servidorEl flux més segur
PKCE (+ Auth Code)Apps mòbils i SPASense secret al client
Client CredentialsComunicació màquina-màquinaAPIs entre serveis
Implicit (deprecated)SPA antiguesNo recomanat

Tokens principals: Access Token (accés a recursos, vida curta), Refresh Token (obtenir nous access tokens).

OpenID Connect (OIDC)

Capa d'autenticació sobre OAuth 2.0. Afegeix l'ID Token (JWT) que conté informació de l'usuari autenticat.

  • JWT (JSON Web Token): token auto-contingut amb header, payload i signatura (base64 + HMAC/RSA).
  • UserInfo endpoint: endpoint per obtenir informació addicional de l'usuari.
ProtocolPropòsitFormatÚs típic

Control d'accés: RBAC i ABAC

  • RBAC (Role-Based Access Control): l'accés es basa en rols assignats a l'usuari (Admin, Editor, Lector).
  • ABAC (Attribute-Based Access Control): l'accés es basa en atributs (departament, hora, ubicació). Més flexible però complex.

Docker, Kubernetes i CI/CD

Els contenidors permeten empaquetar una aplicació amb totes les seves dependències per executar-la de manera consistent en qualsevol entorn. Kubernetes orquestra múltiples contenidors a escala.

Docker

Conceptes fonamentals

  • Imatge: plantilla immutable (sistema de fitxers en capes). És el "motlle" del contenidor.
  • Contenidor: instància en execució d'una imatge. Aïllat del SO amfitrió.
  • Dockerfile: fitxer de text amb les instruccions per construir una imatge.
  • Registry: repositori d'imatges (Docker Hub, Harbor, etc.).
  • Volume: emmagatzematge persistent que sobreviu al cicle de vida del contenidor.
# Dockerfile exemple per a una app Java
FROM eclipse-temurin:17-jre-alpine
WORKDIR /app
COPY target/app.jar app.jar
EXPOSE 8080
CMD ["java", "-jar", "app.jar"]

Comandes essencials

docker build -t myapp:1.0 .       # Construeix imatge
docker run -p 8080:8080 myapp:1.0  # Executa contenidor
docker ps                            # Llistar contenidors actius
docker exec -it <id> /bin/sh        # Shell dins contenidor
docker logs <id>                    # Logs del contenidor

Kubernetes (K8s)

Sistema d'orquestració de contenidors. Automatitza el desplegament, escalat i gestió d'aplicacions en contenidors.

RecursDescripció
PodUnitat mínima. Un o més contenidors que comparteixen xarxa i emmagatzematge.
DeploymentGestiona un conjunt de Pods. Defineix rèpliques, estratègia de rollout.
ServiceExposa un conjunt de Pods com a endpoint de xarxa estable.
IngressGestiona l'accés HTTP/HTTPS extern als serveis del clúster.
ConfigMapConfiguració no sensible com a parells clau-valor.
SecretDades sensibles (contrasenyes, tokens) codificades en base64.
NamespaceAïllament lògic dins el clúster (entorns dev/staging/prod).

CI/CD amb Jenkins

CI/CD (Continuous Integration / Continuous Deployment) automatitza el cicle de construcció, proves i desplegament.

  • Pipeline: definició del flux de CI/CD en un fitxer Jenkinsfile.
  • Stage: cada fase del pipeline (Build, Test, Deploy).
  • Agent: màquina o contenidor on s'executa el pipeline.
// Jenkinsfile bàsic
pipeline {
  agent any
  stages {
    stage('Build') {
      steps { sh 'mvn clean package' }
    }
    stage('Test') {
      steps { sh 'mvn test' }
    }
    stage('Deploy') {
      steps { sh 'docker build -t myapp . && docker push myapp' }
    }
  }
}

Gestió de biblioteques i metadades

En l'àmbit universitari, la gestió de la informació bibliogràfica requereix formats estàndard de metadades, sistemes de repositoris i gestors de continguts especialitzats.

Formats de metadades bibliogràfiques

FormatDescripcióÚs
MARC21Machine-Readable Cataloging. Format numèric de camps i subcamps (p.ex. 245 = títol). Estàndard de biblioteques.Catàlegs de biblioteques (Millennium, Koha)
Dublin Core15 elements simples (title, creator, subject, description…). Estàndard del W3C. Molt usat en OAI-PMH.Interoperabilitat entre repositoris
METSMetadata Encoding and Transmission Standard. Estructura XML per empaquetar objectes digitals complexos.Preservació i intercanvi d'objectes digitals

💡

Per a l'examen: recorda la jerarquia d'abstracció: MARC21 és el format tradicional de catalogació; Dublin Core és el mínim comú denominador per a interoperabilitat; METS/MODS/PREMIS són per a objectes digitals complexos i preservació.

Sistemes de gestió de biblioteques

  • DSpace: repositori institucional open source. Usat per universitats per dipositar tesis, articles, etc. Usa Dublin Core i METS internament.
  • OJS (Open Journal Systems): gestió de revistes electròniques d'accés obert. Desenvolupat per PKP.
  • Drupal i WordPress: gestors de continguts generals, adaptables a webs bibliotecàries.

OAI-PMH

Protocol per a la recol·lecció de metadades (Open Archives Initiative Protocol for Metadata Harvesting). Permet a repositoris exposar metadades en Dublin Core per ser recol·lectades per agregadors (Europeana, Recolecta).

Preservació digital

Garantir l'accés a llarg termini als objectes digitals. Principis de l'OAIS (Open Archival Information System):

  • SIP (Submission Information Package): informació rebuda del productor.
  • AIP (Archival Information Package): informació emmagatzemada a l'arxiu.
  • DIP (Dissemination Information Package): informació enviada al consumidor.

Administració electrònica

L'administració electrònica permet als ciutadans i les institucions relacionar-se amb l'administració pública per via telemàtica. A Catalunya, l'AOC (Administració Oberta de Catalunya) ofereix els serveis compartits de la Generalitat.

Serveis de l'AOC

ServeiDescripció
Via ObertaPlataforma d'interoperabilitat. Permet a les administracions consultar dades de ciutadans sense demanar documentació (padró, títols acadèmics, dades tributàries...).
e-NOTUMSistema de notificacions electròniques. Substitut de les notificacions en paper.
VALIDValidació i autenticació d'identitat digital (certificats, idCAT Mòbil).
e-TRAMTramitació electrònica. Permet iniciar tràmits administratius en línia.
Porta-signaturesGestió del flux de signatura electrònica de documents.
Seu electrònicaPunt d'accés als serveis electrònics d'una administració. Té efectes jurídics.

Signatura digital

Garanteix autenticitat (qui ho ha signat), integritat (no s'ha modificat) i no-repudi (el signant no pot negar-ho). Usa criptografia asimètrica (clau pública/privada).

  • Certificat digital: vincula una identitat amb una clau pública. Emès per una CA (Certification Authority).
  • idCAT: certificat digital de la Generalitat de Catalunya per a ciutadans.

Permisos

Cada fitxer té permisos per a tres categories: propietari (u), grup (g) i altres (o). Cada categoria té tres bits: r (lectura=4), w (escriptura=2), x (execució=1).

# -rwxr-xr-- significa:
# propietari: rwx (7), grup: r-x (5), altres: r-- (4) → 754
chmod 755 script.sh     # Propietari rwx, grup i altres r-x
chown user:group fitxer # Canvia propietari i grup
ls -la                  # Llista amb permisos

import org.openqa.selenium.*;
import org.openqa.selenium.chrome.*;

public class ExempleProva {

    WebDriver driver = new ChromeDriver();
    driver.get("https://www.ub.edu");

    // Trobar element per CSS selector

    cercador.sendKeys("biblioteca");

Usa sempre esperes explícites (WebDriverWait). Les implícites (driver.manage().timeouts().implicitlyWait()) s'apliquen globalment i poden enmascarar problemes de rendiment. Mai barregis les dues.

Page Object Model (POM)

Patró de disseny que encapsula els elements i accions de cada pàgina en una classe. Fa les proves més mantenibles i llegibles.

public class PaginaLogin {

  private By contrasenya = By.id("pwd");
  private By botoLogin  = By.cssSelector("button[type='submit']");

  public PaginaLogin(WebDriver d) { this.driver = d; }

  public void login(String u, String p) {
    driver.findElement(usuariField).sendKeys(u);
    driver.findElement(contrasenya).sendKeys(p);
    driver.findElement(botoLogin).click();
  }
}

POO a PHP

class Usuari {
    private string $nom;
    private string $correu;

    public function __construct(string $nom, string $correu) {
        $this->nom    = $nom;
        $this->correu = $correu;
    }

    public function getNom(): string { return $this->nom; }

    public function __toString(): string {
        return "{$this->nom} <{$this->correu}>";
    }
}

$u = new Usuari("Anna", "anna@ub.edu");
echo $u; // Anna <anna@ub.edu>

Connexió a bases de dades: PDO

PDO (PHP Data Objects) és l'API recomanada per accedir a bases de dades. Suporta múltiples SGBD i protegeix contra injecció SQL amb sentències preparades.

try {
    $pdo = new PDO("mysql:host=localhost;dbname=ub", $user, $pass);
    $pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);

    // Sentència preparada (evita SQL Injection)
    $stmt = $pdo->prepare("SELECT * FROM usuaris WHERE id = :id");
    $stmt->execute([':id' => $id]);
    $usuari = $stmt->fetch(PDO::FETCH_ASSOC);
} catch (PDOException $e) {
    error_log($e->getMessage());
}

Seguretat bàsica a PHP

  • SQL Injection: sempre sentències preparades amb PDO.
  • XSS (Cross-Site Scripting): htmlspecialchars() per escapar sortida HTML.
  • CSRF: tokens aleatoris als formularis per verificar l'origen.
  • Passwords: password_hash() i password_verify() (mai MD5/SHA1 directe).

Mes 1 · Eines de dev
JMeter i SoapUI

JMeter és l'eina estàndard per a proves de càrrega i rendiment. SoapUI és especialitzada en proves de serveis web (SOAP i REST). Ambdues permeten automatitzar i validar el comportament dels sistemes sota pressió.

Apache JMeter

Aplicació Java de codi obert per a proves de rendiment. Simula múltiples usuaris concurrents fent peticions HTTP, JDBC, JMS, etc.

Conceptes clau

  • Thread Group: defineix el nombre d'usuaris virtuals, el temps de pujada (ramp-up) i les iteracions.
  • Sampler: tipus de petició (HTTP Request, JDBC Request, FTP...). El cor de cada prova.
  • Listener: recull i visualitza els resultats (View Results Tree, Aggregate Report, Graph Results).
  • Assertion: valida que la resposta compleix una condició (codi HTTP 200, text a la resposta...).
  • Timer: afegeix retards entre peticions per simular comportament humà.
  • Config Element: configura paràmetres reutilitzables (HTTP Header Manager, CSV Data Set).

Mètriques de rendiment

MètricaDescripció
ThroughputPeticions per segon que el sistema processa
Response TimeTemps des que s'envia la petició fins rebre la resposta
LatencyTemps fins rebre el primer byte de resposta
Error RatePercentatge de peticions fallides
Percentil 90/95/99Temps de resposta que el 90/95/99% de les peticions no supera

SoapUI

Eina de proves per a serveis web. Suporta SOAP, REST, JMS i JDBC. Permet crear suites de proves complertes.

Funcionalitats principals

  • Importar WSDL/Swagger: genera automàticament les peticions a partir de la definició del servei.
  • Mock Services: simula un servei web per provar clients sense el servei real.
  • Assertions: valida codis de resposta, contingut XPath/JSONPath, temps de resposta.
  • Test Suites: organitza casos de prova en suites reutilitzables i encadenades.
  • Data Driven Testing: executa proves amb múltiples jocs de dades des d'un Excel o fitxer de text.
💡

Per a l'examen: JMeter = proves de rendiment i càrrega (quants usuaris aguanta el sistema). SoapUI = proves funcionals de serveis web (la resposta és correcta). Postman és similar a SoapUI però més modern i orientat a REST.

MySQL és el SGBD relacional open source més popular del món. Usat en entorns web (LAMP stack), WordPress, Drupal i moltes aplicacions universitàries. Molt similar a Oracle en SQL estàndard, amb algunes diferències importants.

Diferències clau respecte a Oracle

CaracterísticaMySQLOracle
LlicènciaGPL / ComercialComercial
Auto-incrementAUTO_INCREMENTSEQUENCE + trigger
Límit de filesLIMIT nROWNUM <= n / FETCH FIRST n
ProceduralStored Procedures (MySQL SQL)PL/SQL (molt més ric)

Engines d'emmagatzematge

  • InnoDB (per defecte): suporta transaccions ACID, claus foranes, row-level locking. Usar sempre.
  • MyISAM (legacy): sense transaccions ni FK. Més ràpid en lectures simples. No recomanat per a aplicacions noves.

Creació de taules i relacions

CREATE TABLE usuaris (
  id       INT          NOT NULL AUTO_INCREMENT,
  nom      VARCHAR(100) NOT NULL,
  correu   VARCHAR(200) UNIQUE NOT NULL,
  creat_at TIMESTAMP    DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

CREATE TABLE cursos (
  id         INT NOT NULL AUTO_INCREMENT PRIMARY KEY,
  titol      VARCHAR(200),
  usuari_id  INT,
  FOREIGN KEY (usuari_id) REFERENCES usuaris(id)
      ON DELETE CASCADE ON UPDATE CASCADE
) ENGINE=InnoDB;

Índexs

CREATE INDEX idx_correu ON usuaris(correu);
CREATE UNIQUE INDEX idx_nom_curs ON cursos(titol);

Mes 2 · Ofimàtica
Microsoft Office: Word, Excel i PowerPoint

El paquet Microsoft 365 és l'estàndard en entorns universitaris i administratius. Cal conèixer les funcionalitats avançades, no només les bàsiques, ja que l'examen pot avaluar l'ús professional de les eines.

Microsoft Word: funcions avançades

  • Estils i temes: formatat consistent per generar taules de continguts automàtiques.
  • Taula de continguts: generada automàticament a partir dels estils Títol 1, 2, 3.
  • Combinació de correspondència (Mail Merge): genera múltiples documents personalitzats a partir d'una llista de dades (Excel, CSV).
  • Control de canvis (Track Changes): registra les modificacions de cada autor. Essencial en documents col·laboratius.
  • Camps de fórmula: permet fer càlculs simples a taules de Word.
  • Macros (VBA): automatitzen tasques repetitives. Word Basic o VBA (Visual Basic for Applications).

Microsoft Excel: funcions essencials

Funcions de cerca i referència

=VLOOKUP(valor, rang, col, FALS)     -- Cerca vertical (obsolet)
=XLOOKUP(valor, rang_cerca, rang_ret) -- Modern, bidireccional (Excel 365)
=INDEX(rang, MATCH(val, col, 0))      -- Alternativa clàssica potent

Funcions condicionals i de text

=COUNTIF(rang, criteri)
=SUMIF(rang, criteri, rang_suma)
=IF(condició, si_cert, si_fals)
=IFS(c1, v1, c2, v2, ...)            -- Múltiples condicions (Excel 365)
=CONCATENATE(A1, " ", B1)            -- o bé =A1&" "&B1

Taules dinàmiques (Pivot Tables)

Permeten resumir, agrupar i analitzar grans conjunts de dades amb drag & drop. Elements clau: Files, Columnes, Valors (agregació), Filtres.

Macros i VBA a Excel

Microsoft PowerPoint: bones pràctiques

  • Patró de diapositives (Slide Master): defineix el disseny consistent de totes les diapositives.
  • Animacions vs Transicions: animacions afecten elements de la diapositiva; transicions afecten el pas entre diapositives.
  • Notes del presentador: text visible al presentador però no al públic.
  • Exportació: PDF, vídeo MP4, imatges PNG/JPEG.
Mes 2 · Ofimàtica
SharePoint, Teams i Planner

La suite Microsoft 365 per a la col·laboració empresarial. SharePoint gestiona documents i intranets; Teams centralitza la comunicació; Planner organitza tasques. Tots integrats entre si.

Microsoft SharePoint

Plataforma de gestió documental i intranet corporativa. Base tecnològica de Microsoft 365.

  • Sites (llocs): Team Sites (per a equips) i Communication Sites (per a intranets públiques).
  • Biblioteques de documents: emmagatzematge de fitxers amb versionat, metadades i permisos granulars.
  • Llistes: bases de dades lleugers (com fulls Excel col·laboratius). Suport a Power Automate per a fluxos de treball.
  • Permisos: heriten del site per defecte. Poden trencar-se i personalitzar-se per biblioteca, carpeta o document.
  • Power Automate: automatització de processos (aprovacions, notificacions) sense programar.

Microsoft Teams

Hub de col·laboració que integra xat, videoconferència, fitxers (SharePoint) i aplicacions.

  • Equips (Teams) i canals: un equip agrupa persones; els canals segmenten la comunicació per tema.
  • Pestanyes (Tabs): cada canal pot tenir pestanyes amb fitxers, aplicacions, wikis, Planner...
  • Reunions: videoconferències, gravació, transcripció automàtica, pissarres digitals.
  • Integració amb SharePoint: els fitxers d'un canal s'emmagatzemen automàticament a SharePoint.

Microsoft Planner

Eina de gestió de tasques visual (tipus Kanban) integrada a Teams i Microsoft 365.

  • Plans: agrupació de tasques vinculada a un equip de Teams.
  • Cubells (Buckets): columnes del tauler Kanban (p.ex. Per fer, En progrés, Fet).
  • Tasques: dates límit, assignació a membres, etiquetes, fitxers adjunts, comentaris, llistes de verificació.
  • Vistes: Tauler (Kanban), Gràfic (resum visual), Planificació (calendari).
  • Microsoft To Do: integra les tasques de Planner a la vista personal de l'usuari.
💡

Relació clau: SharePoint emmagatzema els fitxers. Teams és la interfície de col·laboració. Planner gestiona les tasques. Power Automate automatitza els fluxos. Tot comparteix el directori d'usuaris d'Azure Active Directory.

Mes 3 · Gestió de continguts
Drupal i WordPress

Drupal i WordPress són els dos gestors de continguts (CMS) open source més populars. WordPress domina el 40% del web; Drupal és preferit en entorns institucionals i universitaris per la seva flexibilitat i seguretat.

WordPress

CMS basat en PHP i MySQL. Arquitectura senzilla i gran ecosistema de temes i plugins.

Arquitectura

  • Core: nucli de WordPress. S'actualitza regularment (seguretat crítica).
  • Temes: controlen l'aparença. Jerarquia de plantilles (template hierarchy).
  • Plugins: afegeixen funcionalitat. Hooks: add_action() i add_filter().
  • WP_Query: API per consultar contingut de la base de dades.
  • REST API: WordPress exposa el seu contingut via REST API (JSON) per a headless CMS.

Taxonomies i tipus de contingut

PagePàgina estàtica (sobre nosaltres, contacte)
Custom Post TypeTipus personalitzat (producte, curs, notícia)
CategoryTaxonomia jeràrquica per a posts

Drupal

CMS basat en PHP amb arquitectura modular avançada. Preferit per universitats, governs i institucions per la seva granularitat de permisos i extensibilitat.

Conceptes clau de Drupal

  • Nodes: unitat bàsica de contingut (equivalent als posts de WordPress).
  • Content Types: tipus de node amb camps personalitzats (Field API).
  • Modules: equivalent als plugins de WP. Drupal té milers de modules contrib.
  • Blocks: regions de la pàgina que mostren contingut dinàmic.
  • Views: mòdul (inclòs al core) per crear llistats, taules i mapes de contingut sense codi.
  • Entitats: abstracció de tot el contingut (nodes, usuaris, termes de taxonomia, blocs).
  • Drush: eina de línia de comandes per a Drupal (com artisan a Laravel).

Mes 3 · Biblioteca
Repositoris: DSpace

DSpace és la plataforma de repositori institucional open source més usada al món. Permet a universitats i centres de recerca organitzar, preservar i donar accés obert a la seva producció intel·lectual: tesis, articles, informes, dades de recerca.

Arquitectura de DSpace

  • Comunitats: nivell superior d'organització (per exemple, Facultats o Departaments).
  • Col·leccions: dins d'una comunitat, agrupa items del mateix tipus (Tesis, Articles).
  • Items: element bàsic. Conté metadades + un o més bitstreams (fitxers).
  • Bitstreams: fitxers físics associats a un ítem (PDF, imatge, dataset).
  • Bundles: agrupació de bitstreams per tipus (ORIGINAL, THUMBNAIL, LICENSE).

Flux de submissió (Workflow)

  1. L'autor envia l'ítem a una col·lecció.
  2. El sistema aplica el workflow configurat (0, 1, 2 o 3 passos d'aprovació).
  3. Revisors (editors, curadors) accepten o rebutgen la submissió.
  4. L'ítem s'instal·la al repositori i s'assigna un Handle persistent (URI permanent).

Identificadors persistents

  • Handle: sistema d'identificadors persistent usat per defecte a DSpace (ex: http://hdl.handle.net/10490/123).
  • DOI: Digital Object Identifier. Estàndard acadèmic. DSpace pot integrar-se amb CrossRef/DataCite per assignar DOIs.

Metadades a DSpace

DSpace usa Dublin Core com a esquema de metadades per defecte, en format qualificat (dc.title, dc.contributor.author, dc.date.issued...). Suporta esquemes addicionals (MARC, DIM).

OAI-PMH i interoperabilitat

DSpace exposa les seves metadades via OAI-PMH permetent que recolectors com Recolecta, OpenAIRE o Europeana indexin el contingut automàticament sense intervenció manual.

DSpace 7: arquitectura moderna

DSpace 7 (la versió actual) separa el backend (Spring Boot REST API) del frontend (Angular). Permet integrar DSpace en qualsevol aplicació via REST API.

Mes 3 · Biblioteca
Revistes electròniques: OJS

OJS (Open Journal Systems) és la plataforma de publicació de revistes científiques d'accés obert més usada al món. Desenvolupada per PKP (Public Knowledge Project). La UB la usa a revistes.ub.edu.

Funcionalitats principals

  • Gestió del flux editorial complet: des de la submissió de l'autor fins a la publicació.
  • Revisió per parells (peer review): doble cec, cec simple o obert. Gestió de revisors.
  • Publicació per fascicles i volums: organització temporal del contingut.
  • Metadades estàndard: Dublin Core, CrossRef (DOI), DOAJ, PKP PN (preservació).
  • Estadístiques d'ús: integra COUNTER/SUSHI per a informes estandarditzats d'accés.
  • Plugins: extensible. Plugins per a ORCID, Crossref, Google Scholar, etc.

Rols a OJS

RolFuncions
Administrador del llocGestiona totes les revistes de la instal·lació
Director/a de revistaConfigura la revista, gestiona equip editorial
Editor/aGestiona el flux de cada article
Revisor/aAvalua articles assignats
Autor/aEnvia articles i segueix l'estat
Lector/aAccedeix al contingut publicat

Flux de publicació

  1. Submissió: l'autor carrega el manuscrit i omple les metadades.
  2. Revisió editorial: l'editor assigna revisors externs.
  3. Revisió: els revisors emeten un dictamen (acceptar, revisar, rebutjar).
  4. Edició: correcció d'estil i maquetació.
  5. Producció: generació del PDF i galerades finals.
  6. Publicació: assignació a un fascicle i publicació.

Accés obert i llicències

OJS suporta el model Gold Open Access (accés immediat lliure) i integra llicències Creative Commons. Pot configurar-se amb embargaments per a models híbrids.

Mes 3 · Biblioteca
Preservació digital

La preservació digital garanteix l'accés a llarg termini als objectes digitals, malgrat l'obsolescència tecnològica, la degradació dels suports i els canvis de format. És un repte estratègic per a biblioteques i arxius.

Amenaçes a la preservació digital

  • Obsolescència de format: un format (p.ex. WordPerfect) deixa de tenir programari de lectura.
  • Obsolescència de suport: el suport físic (cinta magnètica, disquet) no es pot llegir.
  • Bit rot (degradació de bits): errors físics a l'emmagatzematge que corrompen silenciosament els fitxers.
  • Desastres: incendis, inundacions, fallades de sistema.

El model OAIS (ISO 14721)

Open Archival Information System. Marc de referència estàndard per a sistemes d'arxiu digital.

PaquetAcrònimDescripció
Submission Information PackageSIPEl que envia el productor a l'arxiu
Archival Information PackageAIPEl que l'arxiu emmagatzema internament
Dissemination Information PackageDIPEl que l'arxiu lliura al consumidor

Estratègies de preservació

  • Migració: convertir el contingut a formats nous i oberts (PDF/A, TIFF, WAV).
  • Emulació: recrear l'entorn tecnològic original per executar el programari antic.
  • Replicació: còpies redundants en localitzacions geogràficament distribuïdes. Regla 3-2-1: 3 còpies, 2 suports, 1 fora de les instal·lacions.
  • Normalització: convertir a formats estàndard en el moment d'ingestió.

VídeoMKV/FFV1MP4 (amb pèrdua)

Checksum i integritat

Un checksum (MD5, SHA-256) és una empremta digital del fitxer. Permet detectar el bit rot comparant el checksum calculat amb l'original. PREMIS emmagatzema els checksums com a part de les metadades de preservació.
